使用JCEF开发浏览器

2021-02-12 JCEF 0 评论 字数统计: 3.2k(字) 阅读时长: 14(分)

简介

Java Chromium Embedded Framework 嵌入式框架，一个简单的框架，用于使用Java编程语言在其他应用程序中嵌入基于Chromium的浏览器

JCEF官方项目地址：https://bitbucket.org/chromiumembedded/java-cef/src/master/

JCEF简单的使用文档：http://www.xuanyimao.com/jcef/index.html#jcef

JCEF API 开发文档：http://www.xuanyimao.com/jcef/doc/index.html

为什么要使用JCEF

  • BS结构的优点在于,软件维护和升级方便,只要能上网就能升级了,而且在很多地方都能够直接使用软件,而不需要安装,前提要能登陆到服务器。
  • CS结构缺乏通用性，具有较大的局限性，维护和管理的难度大，但是CS结构的交互性强，而且使用CS能与本地的硬件进行交互（如小票打印机、银联pos机、钱箱等……）
  • 虽然BS结构有诸多优点，但当遇到需要与硬件进行交互时，并不能完全满足业务需求，故而采用了BS+CS结合的方式，希望能同时具有CS结构的交互性与BS的灵活性，使用JxBrowser后，客户端相当于一个带有谷歌浏览器内核的定制浏览器，负责与硬件（如打印机、钱箱、POS机等）的交互，同时负责业务网页的显示，而业务的处理仍是在服务端后台。

  2. 新建lib文件夹，加入jcef相关jar包，并添加到构建路径中 build path—Add to Build path

Js调用Java代码

基本流程：

  1. 首先需要通过调用 CefMessageRouter.create(); 方法创建查询路由对象 CefMessageRouter
  2. 使用路由对象调用 addHandler(CefMessageRouterHandler handler, boolean first)去添加一个处理器
  3. 其中参数一，通过 new MessageRouterHandler() 来创建handler实例，MessageRouterHandler类需要自己创建
    • 使 MessageRouterHandler 类继承自 CefMessageRouterHandlerAdapter 类
    • 因为 CefMessageRouterHandlerAdapter 类是 CefMessageRouterHandler 类的子类，继承父类的方法
  4. 通过 MessageRouterHandler类 中复写 onQuery 方法来执行处理 js 的调用
  5. 页面脚本使用 window.cefQuery 方法来触发上面所复写的 onQuery 函数，执行相关处理返回给页面
    • 注意：可以通过 CefMessageRouterConfig 对象配置页面所使用的查询函数名(默认是：cefQuery)
  6. 最后用客户端对象 client 调用添加路由查询 的方法 addMessageRouter(CefMessageRouter msgRouter) 添加查询路由对象

提醒：

了解基本流程后，通过回调函数去返回数据，如果交互步骤比较多，js代码会嵌套很多层，而且好像只能传字符串类型数据

针对动态加载jar包需求提出的思路：

做成循环下载，就是根据提供响应返回的时间戳和jar包清单数组对象的json数据。把时间戳保存到配置文件，每次启动应用，就发请求检测时间戳是否相同，不同的话，就把jar包清单数组拿出来都放到数组里面，然后去循环去调用下载方法去下载更新，再更新配置文件中的时间戳。

编译32位的CEF二进制文件

编译所需软件

  • CMake 2.8.12.2或者更新的版本（我用的免装版，bin目录要手动配置环境变量）
  • Git
  • Visual Studio 2015（官网推荐2015，原因是开发者使用的就是2015版本，实测VS2010和VS2013编译均会报错）安装时别默认，默认不安装Visual C++，勾选"Visual C++"和"通用windows开发工具"下的"tools和Windows 10 SDK"安装。
  • Java 1.8 32位 （官网写的是1.7或者1.8，实际上使用1.7编译java部分会有语法错误，可以修改源码规避，建议使用1.8，编译64位版本使用64位的jdk）
  • Python 2.6或者更新的版本（不支持Python3）

  1. 克隆官方项目到本地任意文件路径

  2. 在克隆项目下创建新目录jcef_build（不可随意命名）

  3. 打开DOS进入到新目录中，执行一下命令：（如果需要编译64位的话 命令换成这样：cmake -G “Visual Studio 14 Win64” …）

    这个过程会自动去下载当前Jcef对应的CEF的版本，若果下载不成功可以自己下载之后放到/java-cef/third_party/cef目录中。CEF版本号可以从源码目录下的CMakeLists.txt文件中找到

    CEF下载地址：http://opensource.spotify.com/cefbuilds/index.html (经常打不开，我翻墙才能稳定打开)

  4. 命令执行完后会在jcef_build目录生成一个C++项目

  5. 使用VS打开jcef_build中的jcef.sln文件，选择"生成–>配置管理器–>修改为Release和Win32"，然后选择"生成–>生成解决方案"，然后等待构建完成。

  6. 使用dos进入到源码目录的tools目录里，执行命令：compile.bat win32

  7. 同样是tools目录，执行run.bat win32 Release simple或者run.bat win32 Release detailed，浏览器正常运行，编译成功。

  8. 打包，tools目录下执行 make_distrib.bat win32，源码目录下会自动创建打包目录binary_distrib，里面包含了api文档，浏览器Demo，使用到的jar和dll以及资源文件

完成项目打成可运行jar包

  1. 先把java程序按照正常打包成jar包，-Djava.library.path需要运行时设置（在代码里实现一个方法，传入你需要设置的路径）
  2. 把jcef二进制文件复制到打包后的jar所在的目录
  3. 通过：java -jar demo.jar 命令，运行demo.jar
